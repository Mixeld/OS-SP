Задание
Разработать две программы ‒ parent и child.
Родительский процесс, используя fork(), запускает дочерний процесс и управляет его вы-
полнением. Дочерний процесс, используя execve()) замещает код родительского процесса про-
граммой child, которая сканирует ввод с клавиатуры и выполняет указанные действия.
Родительский процесс, прежде чем запустить дочерний, создает сокращенную среду
(окружение), которую будет использовать дочерний процесс при замене его кода программой с
диска. Для этого студентом создается файл env, содержащий небольшой набор имен перемен-
ных окружения, передаваемых при вызове execve().
Минимальный набор переменных в файле env должен включать SHELL, HOME, HOSTNAME,
LOGNAME, LANG, TERM, USER, LC_COLLATE, PATH. Формат файла на усмотрение студента.
Перед запуском программы parent в ее окружении студентом создается переменная
CHILD_PATH с именем каталога, где находится программа child. Если переменная CHILD_PATH не
установлена, предполагается, что программа child располагается в текущем каталоге.
Родительский процесс (программа parent) после запуска получает переменные своего
окружения и их значения, установленные оболочкой, сортирует в LC_COLLATE=C и выводит все в
stdout. Затем читает файл env и формирует «сокращенную» среду для дочернего процесса в том
виде, в котором она указывается в системном вызове execve(), используя значения для перемен-
ных из собственной среды. После этого ветвится и выполняет в цикле чтение stdin и выполня-
ет по получению очередного символа следующие действия:
Символ «+»
Порождает дочерний процесс, используя fork(). Дочерний процесс, используя execve(2),
запускает очередной экземпляр программы child. Информацию о каталоге, где размещается
child, parent получает из своего окружения, используя функцию getenv().
Имя программы child (argv[0]) устанавливается как child_XX, где XX порядковый но‒ -
мер от 00 до 99 (номер инкрементируется родителем).
Вторым параметром программы child является строка "+".
Третьим параметром программы child является массив указателей на переменные «сокра-
щенной» среды.
Дочерний процесс выводит свое имя, pid и ppid в stdout. Затем выводит свою среду в
stdout, получая доступ к среде через внешнюю переменную environ.
Символ «*»
Дочерний процесс порождается и функционирует аналогично предыдущему случаю, одна-
ко получает доступ к среде через третий параметр функции main().
Символ «q»
Завершает выполнение родительского процесса после завершения дочернего.
